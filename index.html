AI Desktop Operator

A local AI that controls your Windows desktop. Tell it what to do in plain language and it does it — opens files, runs commands, types text, clicks things, manages email, and more.

It works the way you'd expect. Say "open Downloads" and it opens Downloads. Say "run git status" and it runs it. No wizards, no extra steps. Just chat and it gets things done.

What it is

  • An actual desktop operator It doesn't just chat — it opens files, presses keys, runs commands, and controls your system.
  • Local by default Everything runs on your machine. Your data stays on your machine.
  • Fast where it counts Common actions like opening folders or pressing keys happen instantly without waiting for the AI.

What it can do

The short version — it controls your desktop through chat.

  • Open anything Files, folders, URLs, and apps — by name or full path. It figures out where things are.
  • Run shell commands Full PowerShell and CMD access with a safety filter that blocks dangerous operations.
  • Control keyboard and mouse Press keys, type text, click coordinates, scroll, and take screenshots.
  • Gmail workflows Read threads, draft replies, forward messages, and send with your approval.
  • Manage windows List open windows, focus the one you need, inspect what's running.
  • Automations Set up tasks that run on a schedule and review what happened later.
  • Process control List, inspect, start, and stop processes when you need to.
  • Multi-step tasks It can chain actions together — open something, check the result, act on it.

What makes it different

  • It acts directly No planning phases, no "analyzing your request". It picks a tool and uses it, same as you would.
  • Common things are instant "Open Downloads", "press Enter", "take a screenshot" — these skip the AI entirely and just happen.
  • Shell access that isn't reckless Full PowerShell and CMD, but with a tested safety filter that blocks credential theft, persistence attacks, and destructive commands.

How it works

The simple version:

you say it → it picks a tool → tool runs → done

Same pattern that powers tools like Claude Code — your message goes to an LLM, the LLM picks the right tool, the tool executes, and you see the result. No unnecessary steps in between.

The interface

Chat in the center. Everything else stays out of the way until you need it.

The conversation is the main interface. Workspaces like Gmail, Automations, and Runs are there when you need them. No dashboards, no clutter.

Things you can say

  • "Open my Downloads folder" Opens it instantly. No full path needed — it knows where common folders are.
  • "Run git status in my project" Executes the command and shows you the output right in the chat.
  • "Take a screenshot" Captures the screen and saves it. Fast enough that you don't notice a delay.
  • "Draft a reply to that email" Works through Gmail, prepares the draft, and waits for you to say send.
Current status: this is an active project with a real working product. The desktop operator is solid and genuinely usable today — it can open things, run commands, control the keyboard and mouse, and work through multi-step tasks. Still evolving, still getting better.
