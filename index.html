AI Desktop Operator

A local, chat-first AI desktop operator for Windows that can analyze the screen, reason about state, plan multi-step work, and execute tasks with approval-gated control.

What makes it different

This is not a chatbot wrapper and not a pile of fragile scripts. It is a structured operator system with explicit evidence, task state, recovery logic, and bounded action scope.

What it does

Built to behave like a real operator instead of an uncontrolled automation loop.

Chat-first control

Tasks start from conversation, not from brittle scripts. The desktop UI stays lightweight while the local API remains the main control surface.

Desktop evidence

The system captures and selects bounded screenshot and window evidence so actions can be grounded in what is actually visible.

Scene interpretation

Recent evidence is interpreted into scene, workflow, readiness, and blocked/loading states to support better decisions.

Controlled actions

Mouse, keyboard, process, browser, and command actions are intentionally bounded rather than fully unrestricted.

Approval-gated safety

Higher-risk actions pause for approval instead of silently pushing ahead like every overconfident automation demo on the internet.

Recovery and readiness

The runtime can reason about minimized, hidden, unstable, or wrong-foreground states instead of assuming the desktop is always perfect.

How it works

A simple view of the current architecture without forcing visitors to read an internal handoff document just to stay alive.

3

Use operator state and task lifecycle controls to decide the next bounded step.

4

Approve

Gate sensitive actions behind explicit user approval when required.

5

Act

Execute browser or desktop actions through bounded, observable control primitives.

Why this project is technically interesting

These are the parts that make it more than a UI skin over some tool calls.

Explicit operator lifecycle

Stop, defer, resume, retry, replace, and supersession control are part of the system model, not afterthoughts.

DPI-safe targeting

Per-monitor coordinate mapping connects screenshot space to final input space so actions are not guessed independently by each tool.

Bounded evidence architecture

Desktop evidence is stored, summarized, and selected deterministically for approvals and follow-up reasoning.
