# Skills

## What a skill is in this repo

A skill is a reusable instruction pattern or capability-specific operating guide that helps the operator perform a recurring kind of task consistently.

Examples:

- safe project inspection
- bounded desktop inspection
- approval-aware desktop action preparation
- browser research and synthesis
- final-answer shaping for read-only investigations

## What skills are for

Skills should help with:

- consistency
- clarity
- reuse
- safer execution patterns
- better final answers

## Suggested skill format

A skill file should usually include:

- purpose
- when to use it
- when not to use it
- required inputs/context
- expected workflow
- safety/approval notes
- expected output shape

## Skill design principles

- keep skills narrow and useful
- avoid giant “do everything” skills
- preserve current architecture and constraints
- keep chat primary and internals secondary
- do not widen dangerous autonomy casually

## Naming suggestions

Use descriptive names such as:

- `project_inspection.md`
- `desktop_inspection.md`
- `approval_gated_click.md`
- `browser_research.md`
- `final_answer_synthesis.md`

Current repo-local desktop skills now also include:

- `desktop_recovery.md`
- `window_state_inspection.md`
- `desktop_readiness_check.md`
